vibox is a simple manager for your VirtualBoxes' VMs.

In facts, it's just a wrapper over VBoxManage, but simplier and with completion.

It's free, build with node.js, and made with 

Getting Started

vibox is build with node.js, so you must have it installed on your computer.

Install the module with: npm install vibox

To take the benefits of completion, add the following line to your .profile, or .bash_rc, or whatever :
which vibox > /dev/null && . "$( vibox --initpath )"

Usage

vibox is a cli tools, so you will type some commands, and vibox will do awesome things.

Quick jump: list show control start headless stop

list

vibox list [options]

Options

-s, --state
Display state of VMs
-r, --running
Show running VMs only

The list command show you all the installed VMs on your system.
You can optionnaly show the state of each VM, and you can filter the results to show only the currently running VMs.

control

vibox control <uid|name> <action>

Actions

start
Start the selected VM
headless
Start the selected VM in headless mode (no GUI)
pause
Pause the selected VM
resume
Resume the selected VM
reset
Hard-reset the selected VM
stop
Send an ACPI-stop signal to the selected VM (if ACPI is enabled)
poweroff
Hard-stop the selected VM

The control command allow you to order the selected VM to perform an action.

start

vibox start [options] <uid|name>

Options

-h, --headless
Start the VM in headless mode (no GUI)

The start command is an alias of vibox control <VM> start.
With the headless option is an alias of vibox control <VM> headless.

headless

vibox headless <uid|name>

The headless command is an alias of vibox control <VM> headless.

stop

vibox stop <uid|name>

The stop command is an alias of vibox control <VM> stop.
